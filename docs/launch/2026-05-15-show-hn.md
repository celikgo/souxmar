# Launch-comms event — Show HN + partner outreach (Sprint 17 push 1)

## Why this doc exists

ADR-0017's launch-comms amendment fires when two consecutive
weeks of external bug reports volume reaches zero. The Sprint
15 retro fired the trigger; Sprint 16 retro re-fired (third
zero week). This doc records the execution Sprint 17 push 1
ratifies.

The amendment is **not** a strategy pivot — external-first
remains the headline. The amendment addresses the *visibility
constraint*: zero-volume isn't bug-shape negative-signal, it's
audience-absent negative-signal. Three actions execute:

## Action 1 — Show HN post

**Title:** "Show HN: souxmar — open-source CAD + FEM + AI agent
pipeline (v0.9.4 public alpha)"

**Body (filed at the time of this commit):**

> souxmar is an open-source CAE platform — CAD geometry import,
> meshing, FEM/CFD solving, and an AI agent that can drive any
> of it from a chat interface. The agent runs against your
> Anthropic / OpenAI key (BYOK) or a local Ollama. Apache-2.0;
> a plugin C ABI for third-party physics modules.
>
> The CLI + Python library are usable today (`pip install
> souxmar`). The desktop app is in active development — the
> chat panel works against BYOK; the workbench viewport is
> still scaffolded. The Pro tier (managed-AI proxy + cloud
> sync + paid plugins) is scaffolded but not yet billable.
>
> Most interested in:
> - bugs in the CLI / Python paths
> - feedback on the agent tool contract (frozen v1 at 18 tools)
> - the plugin author experience for anyone trying to write a
>   solver adapter against the C ABI
>
> Repo: https://github.com/souxmar/souxmar
> Docs: https://docs.souxmar.dev
> Triage SLAs: see docs/COMMUNITY.md (P0 24h / P1 48h / P2 5bd /
> P3 2w).

## Action 2 — Partner outreach emails

Five emails to potential partners. Names redacted in this doc
(privacy); the maintainer's send-record stays in the
operational notebook outside the repo.

Each email:

- 3 paragraphs.
- Concrete ask: "30-minute call to talk about whether the
  agent's tool contract fits your team's analysis workflow."
- Specific to the recipient's domain (CFD-heavy for one
  partner, FEM-heavy for two, multiphysics for one,
  research-software-engineering for one).
- Closes with the docs link + a calendar link.

## Action 3 — INFRA_STATUS launch-result line

The next retro (Sprint 17) reports against:

- HN volume (upvotes / comments / external bug reports filed
  in the first 7 days post-Show HN).
- Outreach response rate (replies / 5 sent).
- The combined number against ADR-0017's targets:
  ≥ 20 external bug reports in the first month;
  ≥ 50% of P0/P1s fixed within the SLA.

The metric reports back in Sprint 17's retro under "what to fix"
if the volume stays below 5 reports — at which point ADR-0017's
**second** amendment fires (rethink the channel, not the model).

## What this doc is NOT

- Not a marketing strategy document. The decision to ship
  scaffolds + Show HN + 5 emails is Sprint 17-scoped; it does
  not commit to a Sprint 18+ launch-comms cadence.
- Not a partner list. The recipients are private; this doc is
  the *event record*, not the address book.
- Not a press release. The HN post is plain-text and links to
  the existing docs site; no campaign assets.
